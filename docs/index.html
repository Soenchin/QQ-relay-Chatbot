CAPABILITY SET

多种功能预装。

双模式对话、读图、链接摘要、插件和 WebUI 集成在同一套中继里。

01 / ROUTING

双模式群聊

直调模式仅在 @ 时回复；管道模式保留聊天窗口，并支持主动接话。两种模式按群自由切换。

02 / VISION

可控读图

按群开启；图片压缩、生成短描述并临时保存，超时、超量或出窗后自动清理。

03 / LINKS

链接摘要

B 站、GitHub 与通用网页；过滤 CQ 图片链，做限流和失败兜底。

04 / PLUGINS

三类插件

消息、notice、request。全局和分群开关由 WebUI 管理。

05 / WEBUI

群聊工作台

现代化群设置、读图开关、管道状态、知识库、人设、插件与实时日志都在本机浏览器里。

06 / SAFETY

把边界留在中继层

管道默认只读；只有主人 @ 才提高工具权限。发送图片前校验本地图床路径，失效图片不会吞掉整段文字。

TWO MODES

两种模式，自由切换。

按群配置 direct 或 pipe，适配不同的聊天方式。

DIRECT

直调模式

面向跑团、工具问答或想保持安静的群。只有 @ 机器人时才请求 API。

  • 最多 50 条本地对话历史
  • 主人可用 `!` 管理命令
  • 响应路径短，行为可预期
  • 适合作为默认 fallback
PIPE

管道模式

面向日常聊天群。CLI 会话与最近 30 条消息共同维持语境，4–8 条普通消息后随机接话。

  • 每群独立 session UUID
  • 默认只读工具权限
  • 主人 @ 时才开放写工具
  • 表情包与图片描述按上下文介入

GROUP VISION

读图模式可控。

只给指定的 pipe 群开启。中继会下载、压缩和描述图片，再把短描述写进聊天上下文；超时、超量或出窗图片都会自动清理。

01下载 / get_image 兜底
02压缩与 GIF 首帧
03多模态短描述
04文字进入上下文

EXTENSIBILITY

插件扩展简单直接。

一个异步函数就是一个处理单元。不同事件走不同注册表，配置保存在 JSON 文件里。

MESSAGE PLUGIN

链接摘要已内置

对 Bilibili、GitHub 和普通网页做摘要；过滤 QQ 多媒体/CQ 图片链，同一链接去重，单群限流，抓取失败也有明确反馈。

async def my_plugin(bot, gid, uid, nick, text, is_at): if "关键词" in text: await bot.send_group(gid, "收到") return True # 截胡后续 AI return False

NOTICE

通知事件

戳一戳回戳、入群欢迎等 OneBot notice 都能独立注册和按群开关。

REQUEST

请求事件

加群申请等 request 事件走专门钩子，不和普通聊天消息搅在一起。

QUICKSTART

十分钟，接上群聊。

先准备 OneBot 实现与 Python。需要 pipe 时再额外安装 Claude Code CLI。

  1. 安装 SnowLuma 或 NapCatQQ，并启用 OneBot v11 WebSocket。
  2. 克隆仓库，安装 Python 依赖。
  3. 复制 `.env.example` 为 `.env`，填 API Key、WebSocket 与主人 QQ。
  4. 配置 `GROUP_MODE`，带 `--webui` 启动中继。
# clone & install
git clone https://github.com/Soenchin/QQ-relay-Chatbot.git
cd QQ-relay-Chatbot
pip install -r requirements.txt

# Windows: copy and edit configuration
copy .env.example .env

# start relay + WebUI
py -3.13 -u relay.py --webui
